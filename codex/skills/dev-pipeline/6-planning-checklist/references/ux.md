# Domínio: UX

Itens exemplo para checklist de qualidade de requisitos UX.

## Hierarquia Visual

- São os requisitos de hierarquia visual definidos com critérios mensuráveis? [Clareza, Spec §FR-1]
- É "exibição proeminente" quantificada com sizing/positioning específicos? [Clareza]

## Estados de Interação

- São requisitos de estado de interação (hover, focus, active) consistentemente definidos? [Consistência]
- Estados de loading são especificados para operações assíncronas? [Completude]
- Estados vazios (empty states) têm requisitos definidos? [Cobertura, Gap]

## Acessibilidade

- São requisitos de acessibilidade especificados para todos os elementos interativos? [Cobertura, Gap]
- Navegação por teclado é definida para fluxos críticos? [Cobertura]
- Contraste de cor atende a WCAG 2.1 AA? [Conformidade]
- Leitores de tela têm labels/aria adequados? [Cobertura]

## Edge Cases UX

- O fallback é definido quando imagens falham ao carregar? [Edge Case, Gap]
- Mensagens de erro seguem padrão consistente? [Consistência]
- Comportamento em conexão lenta/offline é especificado? [Edge Case]

## Responsividade

- Breakpoints são definidos com critérios mensuráveis? [Clareza]
- Layout em mobile tem requisitos distintos do desktop? [Cobertura]
- Touch targets atendem tamanho mínimo (~44px)? [Clareza]

## Microinterações

- Animações são especificadas com duração e easing? [Clareza]
- Feedback tátil/visual em ações destrutivas é definido? [Completude]
