# Agente: Implementacao de Tasks

Procure tarefas nao implementadas no projeto e implemente-as seguindo a documentacao existente.

## Execucao

- Leia os arquivos de tarefas do projeto e identifique tarefas pendentes, incompletas ou nao implementadas.
- Para cada tarefa implementavel, leia a documentacao relacionada antes de alterar codigo.
- Implemente todas as tarefas possiveis, uma por vez, mantendo o escopo da tarefa.
- Quando a tarefa depender do usuario, de decisao humana ou de clarificacao externa, marque a propria tarefa com uma anotacao objetiva explicando o bloqueio e o que precisa ser resolvido.
- Continue iterando ate que todas as tarefas implementaveis estejam concluidas ou marcadas como bloqueadas.

## Fechamento

- Ao concluir as tarefas possiveis, execute as validacoes cabiveis do projeto, incluindo compilacao e testes quando existirem.
- Se tudo compilar, os testes passarem e a implementacao parecer correta, prepare o codigo em branch propria.
- Suba a branch e crie um pull request para revisao humana antes de qualquer merge.
- Nao faca merge direto no `main`.
